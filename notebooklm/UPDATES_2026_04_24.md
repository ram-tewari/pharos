### File 3: Data Model (03_DATA_MODEL_AND_MODULES.md)

#### 1. Semantic Summary Format (DocumentChunk section)
**Added**: Exact format specification for code chunk semantic summaries
```
[language] signature: 'docstring.' deps: [imports]
```

**Why**: This is the actual string that gets embedded (not raw code, not JSON metadata). Understanding this format is critical for debugging retrieval quality issues.

**Example**:
```
[python] def authenticate_user(username: str, password: str) -> User:
    'Authenticate a user with credentials.'
    deps: [verify_password, db.query, User]
```

#### 2. Windows Path Normalization Details
**Added**: Explanation of the backslash → forward slash fix
- The bug: Windows paths like `src\\auth\\middleware.py` broke raw.githubusercontent.com URLs
- The fix: `github_uri.replace("\\", "/")` at ingestion time
- Impact: All 3 302 LangChain resources had broken URLs until this fix

#### 3. Branch Reference Rewriting
**Added**: Why non-SHA refs are rewritten to `HEAD`
- Problem: Branch names like "main" don't resolve for all repos (private repos, renamed branches)
- Solution: `HEAD` always resolves to the default branch
- Edge case: If branch is deleted after ingestion, URL will 404 (accepted trade-off)

#### 4. ChunkLink Table Deep Dive
**Added**: Complete section on parent-child chunk relationships

**Purpose**: Hierarchical retrieval for better LLM context
- Parent class chunk provides context for method chunks
- Parent module chunk provides imports and setup
- Sibling chunks show related methods

**Why it matters**: A function like `authenticate_user()` makes more sense when you also see:
- The `User` class it returns
- The `verify_password()` helper it calls
- Module-level imports (`from bcrypt import hashpw`)

**Query-time expansion**: When `strategy="parent-child"`, system fetches parent chunks too, giving LLM 2-3x more context per hit.

### File 4: Ingestion and Search (04_INGESTION_AND_SEARCH.md)

#### 1. Resource vs Chunk Embedding Strategy
**Added**: Detailed explanation of why resource-level embeddings are used

**Current implementation**:
- Query runs on `resources.embedding` (not `document_chunks.embedding`)
- Chunks inherit parent resource's embedding
- `EXISTS (SELECT 1 FROM document_chunks WHERE resource_id = r.id)` filter skips stale rows

**Why resource-level?**
- Performance: One embedding per file vs. 10-50 per file
- Storage: ~10 MB vs. ~150 MB for 3 302 resources
- Query speed: Top-K over 3 302 vectors faster than 50 000 vectors
- Deduplication: Prevents same file from dominating results

**When will chunk-level be implemented?**
- When storage/performance constraints are relaxed
- When per-chunk precision is needed
- For now, resource-level is "good enough" for Ronin

**Performance implications**:
- Current: ~250ms for top-50 resources
- Chunk-level: ~500-800ms estimated (15x more vectors)
- Trade-off: Slightly less precision for 2-3x faster retrieval

#### 2. Test-Path Penalty Details
**Added**: Why +0.10 specifically and how to adjust

**Empirical tuning**:
- Large enough to down-rank tests below production code
- Small enough that tests still appear if they're the only match

**How to adjust**: Change constant in `vector_search_real.py`
```python
distance = distance + 0.10 if '/tests/' in path else distance
```

**Impact**: Without penalty, 8/10 top results were test files. With penalty, 2/10 are tests (only when highly relevant).

#### 3. Windows Backslash Normalization
**Added**: Detailed explanation of the bug and fix

**The bug**: `Path.as_posix()` wasn't being called on Windows
- Paths like `src\\auth\\middleware.py` stored in `github_uri`
- URLs like `https://raw.githubusercontent.com/.../src\\auth\\middleware.py` returned 404

**The fix**: `github_uri = str(Path(file_path).as_posix())` at ingestion time

**Impact**: All 3 302 LangChain resources had broken URLs until re-ingestion

#### 4. Branch Reference Rewriting
**Added**: Complete explanation of the HEAD rewrite

**The problem**: `https://raw.githubusercontent.com/owner/repo/main/file.py` fails for:
- Repos with "master" as default branch
- Private repos (branch names aren't public)
- Repos where branch was renamed or deleted

**The solution**: `https://raw.githubusercontent.com/owner/repo/HEAD/file.py` always resolves to default branch

**Implementation**: In `ast_pipeline.py`, if `branch_reference` is not a 40-char hex SHA, rewrite to `"HEAD"`

#### 5. BLPOP Timeout Math
**Added**: Detailed calculation for staying under Upstash limits

**Current configuration**:
- BLPOP timeout: 9 seconds
- POLL_INTERVAL: 1 second
- Total cycle: 10 seconds
- Requests per day: 86 400 / 10 = 8 640 req/day
- Margin: 10 000 - 8 640 = 1 360 req/day for other operations

**Prior configuration**:
- BLPOP=5s, POLL=0.1s → ~16 941 req/day (exceeded limit)

**How to calculate for different values**:
```
req_per_day = 86400 / (BLPOP_TIMEOUT + POLL_INTERVAL)
```

**Examples**:
- BLPOP=10s, POLL=0s → 8 640 req/day
- BLPOP=8s, POLL=1s → 9 600 req/day
- BLPOP=5s, POLL=5s → 8 640 req/day

**Trade-off**: Longer BLPOP = lower req/day but higher latency. For Pharos, 9s latency is acceptable (embedding tasks are async).

### File 5: API and Deployment (05_API_DEPLOYMENT_OPS.md)

#### 1. M2M Authentication Details
**Added**: How to generate and rotate API keys

**Generation**:
```python
import secrets
api_key = secrets.token_hex(32)  # 64-character hex string
```

**Rotation process**:
1. Go to pharos-cloud-api service → Environment
2. Add/update `PHAROS_API_KEY` with new value
3. Redeploy (or wait for auto-deploy)
4. Update Ronin's config with new key

**Security notes**:
- API key is **not hashed** (it's an env var, not a DB record)
- Grants **full access** to all endpoints (no scoping)
- Rotate if compromised (no revocation list — just change env var)
- For multi-user scenarios, switch to JWT

#### 2. asyncpg Cast Bug Deep Dive
**Added**: Complete explanation of the silent zero-row bug

**The bug in detail**:
```python
# WRONG (writes zero rows with asyncpg)
stmt = text("UPDATE document_chunks SET embedding = :emb::vector WHERE id = :id::uuid")
result = await session.execute(stmt, {"emb": json.dumps(embedding), "id": str(chunk_id)})
# result.rowcount == 0, but no exception raised

# CORRECT (writes 1 row)
stmt = text("UPDATE document_chunks SET embedding = CAST(:emb AS vector) WHERE id = CAST(:id AS uuid)")
result = await session.execute(stmt, {"emb": json.dumps(embedding), "id": str(chunk_id)})
# result.rowcount == 1
```

**Why this happens**: asyncpg's parameter binding doesn't recognize `::type` postfix casts. It treats `:p::vector` as a parameter named `p::vector` (invalid), so the bind fails silently and WHERE clause matches zero rows.

**How to detect**: Always log `result.rowcount` after UPDATE/INSERT:
```python
result = await session.execute(stmt, params)
if result.rowcount == 0:
    logger.error(f"Expected to update 1 row, but updated {result.rowcount}")
```

**Where this was fixed**: `embed_server.py` had this bug. Fixed in commit that added explicit CAST() calls and rowcount logging.

#### 3. BLPOP Timeout Tuning
**Added**: Complete section on environment variable tuning

**The math** (repeated from File 4 for completeness):

- Requests per day = 86 400 seconds / 10 seconds = 8 640 req/day
- Margin: 10 000 - 8 640 = 1 360 req/day for other operations
